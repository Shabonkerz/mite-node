Mite
A Migration Utility

The goal of Mite is to make database migrations so simple and painless that doing migrations is easier than not doing them.

Check out the GitHub repo →

Tenets of Mite

  • SQL is a perfectly good DSL (Domain-Specific Language)
  • Developers know and have tools for SQL.
  • Down migrations are required for expected behavior on conflict resolution.
  • Order is significant in migrations and is enforced.
  • Migrations should be language agonostic. So a utility is appropriate.
  • Database consistency with the migrations should be enforced. We should be aware of altered migrations.

Installation

  1. Download the distributable or build from source.
  2. Add the location of Mite.exe to your PATH variable.

Getting Started

For new projects:

It's best to have your migrations stay in their own folder, so create one for them: mkdir migrations (then cd into that folder).

Run mite init within this directory. This will take you through a short wizard and spit out a config file that looks something like this:

  {
    "database": "mitedevel",
    "host": "localhost",
    "user": "mite",
    "password": "mite",
    "dialect": "mysql",
    "port": 3306
  }

Got it? Proceed to writing your first migration.

For existing projects:

Same steps as above. No, really.

Powering up an existing Mite project:

Make sure that you have an empty database created and are in your scripts directory. Then run mite update and you're good to go.

Writing your first migration

Migrations with Mite are literally just SQL (well, with some comments).

  /* up */
  CREATE TABLE dbo.test
    (
    id int NOT NULL
    )  ON [PRIMARY]
  GO
  /* down */
  drop table dbo.test

Running mite -c will create a template for you. Just fill it in with, yknow, SQL.
