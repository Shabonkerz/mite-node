Mite
A Migration Utility

sql schema migrations so easy you already know how to do them

Check out the GitHub repo →

Tenets of Mite

  • SQL is a perfectly good DSL (Domain-Specific Language)
  • Developers know and have tools for SQL.
  • Down migrations are required for expected behavior on conflict resolution.
  • Order is significant in migrations and is enforced.
  • Migrations should be developemnt stack + language agonostic
  • Database consistency with the migrations should be enforced. We should be aware of altered migrations.

Installation

npm install --global mite

Getting Started

For new projects:

Create a database for your project (if it doesn't exist), and make sure you have credentials for a user with access to it. If you ever want to make use of mite audit this user needs permission to create/delete databases as well (we promise it will only deal with tmp databases).

cd in to the root directory of your project

Run mite init. This will take you through a short wizard and spit out a config file that looks something like this:

  {
    "database": "mitedevel",
    "host": "localhost",
    "user": "mite",
    "password": "mite",
    "dialect": "mysql",
    "port": 3306
  }

Got it? Proceed to writing your first migration.

For existing mite projects:

Same steps as above. No, really. mite init won't overwrite the migrations directory if it already exists, nor will it mess with your database if the _migrations table is already present.

Writing your first migration

Migrations with Mite are literally just SQL (well, with some comments).

  /* mite:up */
  CREATE TABLE `test` (
    id int primary key
  );

  /* mite:down */
  drop table `test`;

Running mite create will create a template for you. Just fill it in with, you know, SQL.
